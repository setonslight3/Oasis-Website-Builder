Build a Next.js 14 + Tailwind CSS MVP called "OASIS" — a student platform combining educational discovery, peer-shared experiences, and fraud detection (Flagr). This is a clickable demo with mock data, NOT a production system. Focus on polished UI and believable data.

## Tech Stack
- Next.js 14 with App Router and TypeScript
- Tailwind CSS configured with custom colors
- Lucide React for icons
- localStorage for persistence (no real backend)
- Mock JSON data in /data folder

## Design System
Configure tailwind.config.js with these colors:
- Primary: #2563EB (blue-600)
- Accent: #10B981 (emerald-500)
- Background: #FFFFFF
- Surface: #F9FAFB
- Text Primary: #111827
- Text Secondary: #6B7280
- Danger: #EF4444
- Warning: #F59E0B

Use Inter font, rounded-xl cards, rounded-lg buttons, shadow-sm on cards, max-w-6xl container.

## Page 1: Landing Page (/)
- Full-screen hero with gradient background (blue to emerald)
- Headline: "One place you will never have to leave."
- Subheadline: "Discover opportunities, share experiences, and stay safe from scams. Built for Nigerian students."
- Two CTAs: Primary "Try Demo" (links to /dashboard), Secondary "Sign In" (links to /login)
- 3 feature cards below: Universal Search, Experience Network, Flagr
- Each card: icon, title, short description
- Subtle fade-in animation on load

## Page 2: Login Page (/login)
- Centered card (max-w-md)
- Logo at top, Email field, Password field
- "Sign In" button (primary, any credentials work)
- "Continue as Guest" link below → /dashboard
- Mock auth: store fake user in localStorage

## Page 3: Dashboard (/dashboard)
- Welcome banner: "Welcome back, [Name]"
- 3 large interactive cards in a grid:
  1. Search - "Find what you need" → /search
  2. Experience Network - "Read student stories" → /search
  3. Flagr - "Check for fraud" → /flagr
- Each card: icon, title, description, "Open" button
- "Recent Activity" section below showing last 3 viewed experiences from localStorage

## Page 4: Universal Search (/search)
- Large centered search bar with magnifying glass icon
- Placeholder: "Search scholarships, admissions, phone numbers..."
- Example chips below (clickable): "Canada scholarship", "FUPRE admission", "08123456789"
- Enter or click chip → /results?q=...
- Recent searches shown below from localStorage

## Page 5: Results Page (/results?q=...)
- Show search query at top: "Results for: [query]"
- Smart routing based on query:
  - If query is phone-like (regex: /^[0-9+\-\s()]{7,}$/) → show Flagr results
  - Otherwise → show Educational results

Educational Results:
- AI Summary card at top with sparkle icon (2-3 sentence mock summary)
- "Related Experiences" section with 2-3 ExperienceCards
- "Related Topics" pill list linking back to search

Flagr Results:
- Large Risk Score card: phone number, risk level (HIGH/MEDIUM/LOW with color), confidence %, warning if reports > 0
- Linked Reports list: 2-3 mock reports with type, description, date, reporter count
- Recommendation box (colored by risk level)

## Page 6: Experience Detail (/experience/[id])
- Back button at top
- Hero with category tag and title
- Full story text (3-4 paragraphs of believable content)
- AI Summary box (2-sentence version + extracted tags as pills)
- Tags displayed prominently
- "Shared by Anonymous Student • 3 days ago"
- "Was this helpful?" thumbs up/down buttons
- "Report this experience" link (UI only)

## Page 7: Submit Experience (/submit)
- Form fields:
  - Title (text input, required)
  - Category (dropdown: Scholarship, Admission, Visa, Accommodation, Job, Other)
  - Story (textarea, 500 char limit with counter)
  - Tips for others (textarea, optional)
- "Submit Experience" button (primary)
- On submit: success modal "Thanks for sharing! Your experience helps other students."
- Store in localStorage, auto-redirect to new experience page after 2 seconds
- Cancel link → /dashboard

## Page 8: Flagr Page (/flagr)
- Hero: "Check before you trust."
- Tabs: Phone | Account Name | Website
- Form changes based on tab (phone input, text + bank dropdown, URL input)
- Common: "Description of incident" textarea
- "Check Now" button
- Results appear below with same risk score UI

## Logic (/lib/search.ts)
- isPhoneQuery(query) — checks if query is phone-like
- searchExperiences(query) — fuzzy match on title, summary, tags, category
- searchFlagged(query) — exact and partial match on target field
- summarizeExperience(exp) — returns mock AI summary string
- extractTags(text) — returns mock tags based on keywords

## Storage (/lib/storage.ts)
- getUser(), setUser(), logout()
- getRecentSearches(), addRecentSearch()
- getUserExperiences(), addUserExperience()
- getViewedExperiences(), addViewedExperience()

## Polish Requirements
- Loading states with skeleton loaders on results
- Empty states with friendly messages
- Custom 404 page: "Lost in the desert"
- Mobile-first responsive design
- Smooth transitions and hover effects
- Toast notifications on submit success
- Proper accessibility (labels, focus rings, semantic HTML)

## Do NOT Build
- Real authentication
- Real database
- Real AI API calls (mock all AI)
- Admin dashboard
- Payment integration
- Email notifications
- Real moderation
- Vector search

4. "Canada scholarship" shows educational results
5. "08123456789" shows Flagr results with risk score
6. Experience detail page renders with AI summary

8. Flagr tabs switch and show results
9. Mobile responsive
10. Navbar on every page except landing

Build it clean, build it fast, build it beautiful.
